Qimah Eyes

The analytics brain behind Qimah - an internal dashboard that turns raw platform data into actionable decisions for the team.

Qimah is an online education platform. Right now, our data lives in 7+ separate systems.
Qimah Eyes pulls it all into one place so we can see what's working, what's broken, and what to do next.

? Why Are We Building This?

Today, checking revenue means opening WooCommerce. Checking student progress means LearnDash. Email health is in Resend. Recruitment is in ClickUp. Video stats are in Bunny CDN. No one has the full picture. Qimah Eyes fixes that - one screen, every morning, all the signals that matter.

~ How Data Flows
WordPress Moyasar Bunny CDN Resend ClickUp PostHog
|
webhooks + nightly API pulls
v
n8n (automation) -> Node.js scripts -> Postgres DB -> Next.js Dashboard
3 Three Layers of the Pipeline
1
Ingestion
Collect Data
Receives real-time webhooks from WordPress (orders, activity, security events) and pulls APIs nightly (Moyasar, Bunny, Resend, ClickUp, PostHog).
n8n - 14 workflows
2
Computation
Crunch Numbers
Nightly scripts that compute streaks, instructor scores, cohort stats, session patterns, and run 20 insight rules + weekly AI analysis.
Node.js + SQL
3
Storage
Serve the Dashboard
33 tables of clean, queryable data. The Next.js dashboard reads from Postgres and presents 9 sections: Revenue, Learning, Retention, and more.
Postgres 16 + Next.js
* What You'll Be Doing
n8n Workflows
Build the automation workflows that receive webhooks and pull data from external APIs. Visual, drag-and-drop - you'll learn fast.
Node.js Scripts
Write the nightly computation scripts - mostly SQL queries wrapped in JS. Streaks, scores, aggregations.
Postgres Queries
Design upserts, write window functions, build aggregations. The heavy lifting happens in SQL, not code.
Dashboard UI
Help build the Next.js frontend that visualizes all of this - charts, tables, real-time pulse indicators.
